# 先行技術調査エージェント

機能を実装する前に、その概念が既に存在するか、何と呼ばれているか、どの OSS ソリューションが利用可能かを素早く特定します。

## このエージェントの役割

1. **概念に名前を付ける** — 本当の技術用語を発見（例：「スムーズなシーク」→「タイムストレッチング」）
2. **リスクをマッピング** — 何が失敗する可能性があるか特定（アーキテクチャ、パフォーマンス、採用）
3. **OSS オプションをリスト** — 既存のソリューションで活用できるものを表示

## 使用時期

### フェーズ 1: 要件定義（クイックチェック）
- ✅ 問題を定義している？
- ✅ 設計フェーズに進む前？
- **使用**: `@prior-art-investigation minimal` → 5-10 分、 ~150 トークン

### フェーズ 2: 設計（深層調査）
- ✅ アーキテクチャ決定が必要？
- ✅ 新しいサブシステムを設計している？
- **使用**: `@prior-art-investigation full` → 20-40 分、 ~500 トークン

### フェーズ 3: 不明な場合
- ✅ 初回利用？
- **使用**: `@prior-art-investigation selector` → 1-2 分（自動ルーティング）

## クイック例

### 例 1: 要件定義フェーズ
```
ユーザー: "@prior-art-investigation minimal ユーザーがリズムパターンでグルーヴを検索できる方法が必要"

エージェント: 
- 実際の概念: 「シーケンスアライメント」または「リズムフィンガープリント」
- Q1 第一原理: 問題は「グルーヴを検索すること」か「類似リズムを見つけること」か？
- Q6 リスク: 
  • 仮定: ユーザーはリズムパターンを理解している
  • リスク: 学習曲線が採用をブロックする可能性
  • 検証: ユーザーインタビューでパターン認識を確認
- 設計フェーズで FULL 調査を推奨
```

### 例 2: 設計フェーズ
```
ユーザー: "@prior-art-investigation full 新しい ML ベースの境界検出システムを設計しています"

エージェント:
- Q1-Q7 完全な調査
- 既存 OSS: librosa、essentia、madmom
- アーキテクチャオプション: リアルタイム vs バッチ
- 先行技術からのパフォーマンスベンチマーク
- 推奨: まず librosa をベースライン、その後 madmom で高度な機能を追加

研究テンプレートを .kiro/specs/[feature]/docs/research.md に保存
```

## コマンド

| コマンド | 効果 | 時間 | トークン |
|---------|------|------|---------|
| `@prior-art-investigation selector` | フェーズを自動検出、適切なプロンプトを実行 | 1-2 分 | ~100 |
| `@prior-art-investigation minimal` | Q1 + Q6 のみ（要件定義） | 5-10 分 | ~150 |
| `@prior-art-investigation full` | Q1-Q7 完全（設計） | 20-40 分 | ~500 |

## 出力

すべての調査は以下を生成：
- ✅ 概念名確認 + 代替名
- ✅ OSS オプションをランク付け
- ✅ リスクマップ（仮定 → 潜在的な障害）
- ✅ 研究テンプレート（Kiro SDD ワークフローで `.kiro/specs/` に保存）

## 統合

### スタンドアロン（VS Code）
```
@prior-art-investigation minimal
```

### Kiro SDD と統合
```bash
# フェーズ 1（要件定義）
/kiro-spec-requirements FEATURE-NAME
# → minimal 調査を実行

# フェーズ 2（設計）
/kiro-spec-design FEATURE-NAME
# → full 調査を実行、.kiro/specs/FEATURE/docs/research.md に保存
```

```javascript
import minimalPrompt from '@ma2tani/prior-art-investigation/prompts/minimal';
// 自分の LLM パイプラインで使用
```

## 主な特徴

🎯 **フェーズ対応** — クイックゲートには minimal を、深い決定には full を実行

🚀 **ゼロ設定** — `@prior-art-investigation` と入力するだけ

📦 **OSS 重視** — 事前にスクリーニングされたオープンソースソリューション

🔄 **Kiro SDD 互換** — 出力を仕様ワークフローに保存

💬 **バイリンガル** — 英語と日本語で動作

## 次のステップ

1. **初回利用？** → `@prior-art-investigation selector` を実行
2. **要件を持っている？** → `@prior-art-investigation minimal [あなたの機能]` を実行
3. **深掘り必要？** → `@prior-art-investigation full [あなたのサブシステム]` を実行
4. **Kiro を使用中？** → 結果は `.kiro/specs/[feature]/docs/research.md` に自動保存
