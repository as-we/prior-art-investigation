# 先行技術調査 — SELECTOR（自動フェーズ判定）

> **目的**: 開発フェーズに基づいて最適なプロンプトへの自動ルーティング
> **役割**: 内部ルーター - 最初にこれを実行して、どのプロンプトを使うかを判定します
> **時間**: フェーズ判定までで 1 分

---

## あなたはどのフェーズにいますか？

次の 3 つの質問に順番に答えてください：

### **質問 1: 機能の定義度**

**あなたの機能は要件で既に名前が付いていますか？**

- ✅ **はい** → 明確な要件名がある → **質問 2 へ**
- ❌ **いいえ** → 要件が曖昧または無名 → **MINIMAL を使用**
- ❓ **不明** → 部分的に定義されている → **MINIMAL を使用**

---

### **質問 2: 利用可能な時間**

**調査に 30 分以上の時間がありますか？**

- ✅ **はい、30 分以上ある** → **質問 3 へ**
- ⚠️ **15-20 分くらい** → **MINIMAL を使用**（5-10 分の迅速チェック）
- ❌ **いいえ、急いでいる** → **MINIMAL を使用**（5-10 分の迅速チェック）

---

### **質問 3: 決定の重要性**

**これは主要なアーキテクチャ決定または新しいコアサブシステムですか？**

- ✅ **はい、重要** → **FULL を使用**（20-40 分の深掘り）
- ⚠️ **どちらかといえば** → **FULL を使用**（大きな決定は慎重に）
- ❌ **いいえ、段階的な変更** → **MINIMAL を使用**（迅速な検証）

---

## 出力: 使用するプロンプト

### **MINIMAL を使用する場合:**
- [ ] 要件定義フェーズ（概念検証が必要）
- [ ] 時間: 5-10 分利用可能
- [ ] スコープ: 迅速な検証のみ（Q1 + Q6）
- [ ] 目標: 問題が実在することを確認、主要リスクを特定

**ファイル**: `minimal.prompt.md`  
**フェーズ**: 要件定義  
**トークンコスト**: ~150

---

### **FULL を使用する場合:**
- [ ] 設計フェーズ（包括的な発見）
- [ ] 時間: 20-40 分利用可能
- [ ] スコープ: 深層調査（Q1-Q7 + OSS マップ）
- [ ] 目標: 命名された概念 + 設計への影響分析

**ファイル**: `full.prompt.md`  
**フェーズ**: 設計  
**トークンコスト**: ~500

## Kiro SDD との統合

**Kiro 仕様駆動開発**を使用している場合：

| フェーズ | プロンプト | Kiro フェーズ | トークン予算 |
|---------|----------|------------|-----------|
| 要件定義 | MINIMAL | フェーズ 1 | ~150 |
| 設計 | FULL | フェーズ 2-3 | ~500 |
| タスク設計 | Task Router | フェーズ 4 | ~75 |
| **合計 SDD コスト** | - | 1-4 | **~725**トークン |

> **ルーティングなし場合**: ~2000+ トークン（非効率な調査）
> **ルーティングあり**: ~725 トークン（**64% 削減**）
